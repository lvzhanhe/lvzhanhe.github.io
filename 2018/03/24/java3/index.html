Generics

A generic method is a method definition having a special type parameter that may be used in place of types in the method.

A generic class is a method definition having a special type parameter that may be used in place of types in the class.

Type arguments can only be reference types. If a primitive type is desired, it should be boxed into a corresponding primitive wrapper class.

A type parameter may be associated with a type bound to specify the class types for which a type parameter is valid. For instance, <Type extends Comparable<Type>>.

Perfect size array

A perfect size array is an array where the number of elements is exactly equal to the memory allocated.

A perfect size array can be created by initializing each element in the array to given values.

E.g., int[] studentIDs = {93, 80, 62, 75, 74, 89, 97};.

A perfect sized array is used when the size of the array will not change throughout the program.

A new array should be constructed if number of elements in it need to be changed.

Oversized array

An oversize array is an array where the number of elements in the array is less than or equal to the memory allocated.

Oversize arrays are good choices when the number of elements in the array will change over time.

Oversize arrays typically use three variables: array reference, current size, and a constant for the array capacity.

When oversize arrays are passed to a method, array reference and the current size are needed. Besides, an oversize array cannot be returned by a method.

Methods updating oversize arrays (do not forget to update the size of array after modifications)

  • Adding elements
  • Removing elements

Linked List

A data structure is way of organizing data, usually used to implement an abstract data type.

The first node is called head. The last node is called tail.

A list traversal algorithm visits all nodes in the list and performs some operation on each node.

A search algorithm returns the first node whose data matches that key, or returns null if a matching node was not found.

Common methods:

  • get(index)
  • set(index, newElement)
  • add(newElement), add(index, newElement)
  • size()
  • remove(index), remove(existingElement)

O notation

Big O notation is a mathematical way to describe the relationship between running time of an algorithm and the input size.

Constant, Logarithmic, Linear, Log-linear, Quadratic, Exponential.

Algorithm runtime analysis often focuses on the worst-case runtime complexity.

Stack and Queue

A stack is an ADT in which items are only inserted on or removed from the top of a stack.

A queue is an ADT in which items are inserted at the end of the queue and removed from the front of the queue.

ListIterator

Common methods:

  • next()
  • nextIndex()
  • previous()
  • previousIndex()
  • hasNext()
  • hasPrevious()
  • add(newElement) Inserts a newElement between the next and previous elements and moves the ListIterator after the newElement.
  • remove() Removes the element returned by the prior call to next() or previous().
  • set(newElement) Cannot call set() immediately after add() or remove(). Replaces the element returned by the prior call next() or previous() with newElement.

Recursion

A recursive method is a method that calls itself.

A recursive method should have at least a base case and a recursive case.

Use recursion to solve math problems:

  • Fibonacci sequence
  • Greatest common divisor

Each method call places a new stack frame on the stack. Deep recursion could fill the stack region and cause a stack overflow.

Searching and algorithms

Linear search starts from the beginning of a list, and checks each element one by one until the search key is found or the end of the list is reached.

Binary search is based on a sorted list. It checks the middle element of the search space until the search key is found ot the search space is empty.

Binary tree

Since “binary” means two, each node has up to two children.

Some definitions:

  • Leaf: A tree node with no children.
  • Internal node: A node with at least one child.
  • Left child
  • Right child
  • Parent
  • Ancestor
  • Root: The top node. The one that does not have parent.
  • Edge: The link from a node to its child.
  • Depth: The number of edges from the root to the node.
  • Level: All nodes with the same depth form a tree level. All nodes with depth 1 form level 1.
  • Height: The largest depth of any node. A tree with just one node has height 0.

A max-heap is a binary tree that maintains the simple property that a node’s key is greater than or equal to its childrens’ keys.

An insert into a max-heap starts by inserting the node in the tree’s lowest level, and then keeps swapping the node with its parent until no max-heap property violation occurs.

A remove from a max-heap replaces the node with the last level’s last node, and keeps swapping that node with its greatest child until no max-heap property violation occurs.
